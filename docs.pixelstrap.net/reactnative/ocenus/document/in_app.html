In-App Payment
  1. There's a significant amount of setup required for testing in app purchases successfully, including registering new app IDs and store entries to use for testing in both the Play Developer Console and App Store Connect. Both Google Play and the App Store require developers to configure an app with in-app items for purchase to call their in-app-purchase APIs. Both stores have extensive documentation on how to do this, and we've also included a high level guide below.
  2. In-App Purchase (App Store) - https://developer.apple.com/in-app-purchase/
  3. Google Play Billing Overview - https://developer.android.com/google/play/billing/billing_overview

For Android:

  1. Create a new app in the (Play Developer Console) - https://play.google.com/apps/publish/
  2. Sign up for a merchant's account in the Play Developer Console.
  3. Create subscriptions in the Play Developer Console available for purchase in the app. The example assumes the following SKU IDs exist:

    • Weekly plan with subscription id : weeklyplan_330.
    • Monthly plan with subscription id : monthlyplan_826.
    • Lifetime plan with subscription id : lifetimeplan_2375.


  • Make sure that all the subscriptions are set to ACTIVE.

    For IOS:

    1. Create a new app in the (Apple Developer Console) - https://appstoreconnect.apple.com/apps
    2. Go to subscription tab and create subscriptions in the Apple Developer Console available for purchase in the app.
    3. Now give name and click on create button.
    4. Now click on availablity to set country in which you want to show in-app.
    5. Set availablity as per your choice in which country you want to show subscription.
    6. Now click on Add Subscription Price to add price for your in-app.
    7. Now click on choose and select price for in-app and click on next.
    8. Now if you want to change currency for 1 country or multiple then you can click on price and can change or else just click on next.
    9. If you think you still need to do some changes then click on back and do you changes and click on next or else click on confirm.
    10. Now we will add localization so for that click on Add Localization.
    11. Now add Name and Description and click on Add this will display on appstore.
    12. After adding localization it will look like this.
    13. Now we will add Review information for the app store to check our subscription this details are not displayed on the app store it's for the app store itself add a screenshot and review notes so it will be easy for the app store to check our subscription.
    14. After adding all details click on save.
    15. After adding all details in subscription it will look like this.
    16. As we are seeing that still it showing missing metadata so for that we need add localization.
    17. Click on create to create localization.
    18. Now give name and click on create.
    19. If after adding localization if you still see missing metadata then check if you have added duration or not.
    20. Select duration and click on save.
    21. Now our all subscriptions will be in ready to submit which is very important to make our in-app work.
